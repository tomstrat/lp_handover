Hosting

GoDaddy

Go Daddy is Local Pages original hosting environment. It currently holds the most sites to date. The reason Go Daddy was chosen as a hosting platform originally was to accommodate UPUGO, a 3rd party we used at the time. While we still host many of our websites on this platform, new websites use our other two platform choices. Migration plans are in discussion and Go Daddy may become redundant at some point in the future.

Go Daddy is easy to use and tends to hold the user's hand when using the platform. The downside to Go Daddy is every product is packaged and generally more expensive than other alternatives.

Go Daddy offer managed Wordpress hosting, SSLs included and some small level of hosting support. You can purchase more web support if required. Go Daddy also offers a Wordpress maintenance dashboard allowing mass updating of plugins and other support features.

Siteground

Siteground is now Local Pages preferred choice of hosting platform when creating websites in-house.

Siteground prides itself on superior managed Wordpress hosting. Local Pages made the move to Siteground primarily as a test for Fit My Floor, and after use we preferred its simplicity, support and price.

Siteground while less user friendly than Go Daddy, is still simple to use, comes with a very handy live chat and lots of freedom to manage the hosting environment.

Siteground offers managed Wordpress hosting, free SSLs, web support and file manager, plus lots of other developer features. It also boasts very quick speeds for bigger sites.

I have listed below some plugins we use on the large majority of sites and ones I would turn to before others:

  • Better Search Replace

FMF

One of the more complex sites we house with Wordpress and host on Siteground is Fit My Floor.

While I won’t go into detail on how to develop for the site or maintain what is there (they are replacing soon) I will explain its basic infrastructure and any quirks to be aware of.

FMF is hosted on Siteground, the sole reason is to do with space requirements and the ability to access more than just the public_html folder. I would recommend you leave this site on here until they replace it, as you may run into issues putting it onto Godaddy.

FMF’s origin was developed with UPUGO. This means that there are some skeletons in the closet that can appear from time to time. The main one being the strange configuration they set for the shop when FMF wasn’t selling online. Once UPUGO we’re gone we have worked hard to convert their work into the online shop it is today. What this means, is that there are sections of the site that are NOT online shop, and these are here for legacy purposes.

The “Online Shop” section has its own products that are categorised by “real product” on the back end. Without this checked, the product will not be available for purchase, or appear on the “Online Shop section”.

Many of the real products also have a lot of customisation with new fields depending on category and custom styling for the single item pages. I recommend this not be touched or changed, as the code written for it is complicated and can tank the site if messed with.

Database

Directory Generator

This is the tool used to download XML from the current database. Its in a hidden URL location: “database.ad.localpages.co.uk/index.php?target=directorygenerator”.

Its simple to use, simply pick a book to download and choose XML.

DO NOT choose XML + AUI as this setting is in testing and doesn’t do as intended.

There is a guide on SharePoint on how to add this data to the InDesign document found here: Guide

Check All Free Entries

While the current database is still in use you will need to ensure all free entries are “checked” or compilated. The page to do this is in Books -> Select Book -> All Free Entries.

There are check boxes next to all the free entries that need to be ticked then submitted (at the top) for them to be added to the XML download. Many of the books have thousands of these and the task of ticking them all can take a long time. A solution to this would be to use the code below in the console to mass tick them all.

Right click the webpage and select inspect. Navigate to the console tab and paste this code in and submit:

(function() {
    const inputs = document.querySelectorAll("input[type=checkbox]");
    for (let i = 0; i < inputs.length; i++){
        inputs[i].checked = true;
    }
})();

Phone Data

BT

The BT phone data can be found in TC Reports under Sales Call Monitor. BT Phone data is found on their portal. Login credentials can be found with Ollie.

The data retrieved is only based on the landlines of Local Pages, for Sales purposes this is now virtually mute as sales primarily use their mobile phones. That being said this data can still be used to find numbers who called Local Pages or who employees called from the landlines.

This portal can also be used to update the settings of the phones in the office. There are MANY settings all of which can be confusing (I don’t fully understand them all). Despite this the support team for BT Cloud Voice are very helpful and always give me the needed information. Support

Vodafone

The Vodafone data can be found in TC Reports under Sales Call Monitor. Vodafone data is found on their portal. Login credentials can be found with Ollie.

The data retrieved is based on the mobile phones of Local Pages. This is the primary data for Sales. It should be noted that the data can only be taken in 3-6 month intervals and starting and ending in the middle of the month. This is due to the billing period.

It should also be noted that their portal is very temperamental and often breaks or is offline. Also many of the buttons don’t do as intended or don’t work at all.

Allocation

Allocation data and spreadsheets can be found in TC Reports under Allocation.

There are lots of files here, mainly things that are for the current round of allocation. Previous data and spreadsheets can be found in the Archive folder.

The most notable folder for someone taking over is the Tools folder. This folder contains all the necessary information to build a new allocation folder using the data available to us. (this will change entirely when the new database is implemented).

  • Instructions.txt is a verbose list of steps to take, to make a new allocation datasheet.
  • IDS.csv and IDS.xlsx are a list of the IDs for the customers and staff on our current database.
  • Home Book.csv is a list of the customers home books.

LP Signature is a tool desinged to allow easy access for employees to change their signature when marketing deem it necessary. This tool has a front end and can be found here: LPSig.

While this has a front end, in order to change the content of the signature, the back end must be adjusted. The back end consists of HTML CSS and JS. The HTML is written in a very specific way in order to accomodate for email signatures (tables).

Asana Scraper

The Asana Scraper is a very specific tool designed to backup all LPs Asana tasks and data to a spreadsheet for storage.

The scraper is a command line tool and is written for the specific task of backing up LP data. Because of the lack of front end, this tool should only be used by a technical employee who understand Node.js.

Dentons Scraper

The Dentons Scraper is a tool designed to scrape the Dentons.net website for listings.

This tool does see use when Sales want some extra data to parse especially if a new area is added to the books.

This tool is a command line application with no front end. Its also quite a specific scraper for the current version of the Dentons.net site. This should only be used by a technical employee who knows Python / Node.js

The current Dentons.net site has an empty robots.txt (the file stating scrapers allowance. THIS COULD CHANGE and if thats the case then i reccomend you do not scrape unless confident its still ok to do so.

Import Artwork

The Import Artwork tool is a JSX InDesign script designed specifically for pulling in artwork to a document (LBB).

It works by having you choose a folder for it to parse, then it scours the document for the artwork tag, and takes its contents (the name) and finds the corrosponding .eps in the chosen folder.

The method highlighted above, is very specific and would require recoding to change the way it works. This is why its important the process stay the same for this to continue to be used.

This can also be found on my github if needed, but the script is the InDesign applications already and can be taken or backed up there.

The script can be found here Import Artwork

This script saves HOURS of time for the builder of the book and is quite critical for the process to remain smooth. If any facet or aspect of the process changes or the script requires changing or improving, a technical employee will need to have a strong knowledge of Javascript and the Adobe Application API.
